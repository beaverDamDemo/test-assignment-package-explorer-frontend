Test Assignment

Create an Angular application for displaying data about npm packages with the following interface:

  • On startup, the application should send a request to load package data (see the API section).
  • Each package should be displayed as a card:
    • The header must display the package name. If the name is scoped (contains /), the first part (scope) should be highlighted with a different color.
    • The body must display the number of downloads and number of dependencies. If the number exceeds 1000, it should be rounded down to thousands and displayed with the letter “K”.
  • When hovering over a card:
    • Its header should be highlighted.
    • The headers of all cards representing packages that are dependencies of the hovered package should also be highlighted with a different color (for example, tslib is a dependency of @angular/core).
  • The package list must be scrollable.
  • The user should be able to filter packages by name.
  • The user should be able to reload all data using a button or another mechanism.

API

For testing, use the Node.js API: mock-interview-api

GET /packages – returns a list of all packages:

{
        id: string;
        weeklyDownloads: number;
        dependencyCount: number;
      }[]

GET /packages/:id/dependencies – Returns a list of dependency package IDs for the specified package:

string[]

Additional Notes

  • The use of third-party libraries is allowed.
  • The UI design and level of visual detail are up to you (a complex interface with animations, etc., will not be taken into account during evaluation).
